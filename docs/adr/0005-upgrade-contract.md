# The upgrade contract: prompted, machine-scoped, version-stamped

Upgrades are prompted — a throttled one-line notice in the Digest — never silent; `upgrade` swaps the plugin/bundled CLI and migrates local machine state only, and committed artifacts change exclusively through an explicit `migrate` that emits a reviewable diff. Every schema (Team Memory repo format, per-repo scope config) carries a version stamp: newer tools must always read older schemas, while an older tool meeting a newer schema fails loud and falls back to the last-good cached Digest with a staleness warning. We accepted days of version skew across the team (versus auto-update lockstep) because agents consuming silently changed or misread memory is the exact failure mode this product exists to prevent. The backward-read promise is bounded to the current major version — cross-major skew falls back to the stale cached Digest until the person upgrades — so the compatibility surface stays finite at org scale.
