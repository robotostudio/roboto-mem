# Interactive prompts are TTY-gated sugar over flags, rendered by clack

Commands whose inputs teammates can't be expected to memorise — `init`, `promote`, `skill add`, `skill promote` — prompt for missing values when stdin and stdout are both TTYs; provided flags are never re-asked, and non-TTY invocations (the SessionStart hook, CI, scripts) behave byte-for-byte as before, so the digest path can never block a session on a question. Rendering comes from `@clack/prompts`, imported dynamically only once the interactive branch is taken: against ADR 0007's bar, the select/cancel/validation machinery it replaces is exactly the meaningful tested code a dependency must delete, and the cold digest path parses none of it. Measured at adoption (July 2026): +109.5 kB raw (511.5 → 621.0 kB, +21.4%), +27 kB gzip, cold start flat at 0.04s for both `status` and `digest --hook` — the clack code compiles to a memoized lazy initializer unreachable from any non-interactive path. We accepted the raw growth deliberately: this sits at the size that sank zod in ADR 0007, but zod lost to an equivalent 11.5 kB substitute, whereas the substitute here is owning cursor rendering, cancel handling, and ANSI wrapping forever. The bar that binds is cold start (every session start pays it) plus the artifact staying one flat file — both unchanged. Reversal stays contained by design: prompts sit behind a `PromptDriver` interface, so swapping to hand-rolled `node:readline` later touches one factory function, not the orchestration or its tests. Rejected: `consola.prompt` (drags a logging framework alongside citty), `inquirer` (size), raw `node:readline` (rebuilds render/cancel logic we'd own and test forever). `digest`, `sync`, `status`, and `lint` never prompt by design — machine paths stay machine paths. One deliberate asymmetry: when a guided `promote` hits an exact collision, an explicit confirm (default No) may propose overwriting the existing Entry via an internal option no CLI flag reaches — `--force` keeps its narrower tested meaning (near-duplicate dedupe bypass only, never exact-collision bypass), so scripts can never clobber an Entry, while a human answering a prompt can still propose it and the Scope's Owners still review the PR.
