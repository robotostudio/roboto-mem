# A dependency must delete meaningful code and keep the artifact flat

Every dependency ships inside the committed single-file bundle (ADR 0004) and loads on every session start, so the bar is: it must delete meaningful tested code AND leave bundle size and cold-start essentially unchanged. Measured against that bar (June 2026, real tsdown bundles of our exact use cases): no swap clears it today — zod classic +122 kB, gray-matter +329 kB, execa +230 kB, arktype +279 kB all fail on size; tinyexec (+8 kB) and semver libs fail on deletion (they'd replace 12-30 already-tested lines). When new schema surface genuinely arrives (the `migrate` command, v2 formats), **valibot** (+11.5 kB raw / 2.6 kB gzip for our shapes) is the sanctioned validator over zod/mini (twice the size) — chosen by measurement over the org's default Zod preference. `yaml` stays despite being ~234 kB (half the artifact): entries are hand-authored frontmatter, and YAML edge cases are exactly where smaller parsers fail.
